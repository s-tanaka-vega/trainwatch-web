# TrainWatcher 仕様メモ

`index.html` 単体で動くダッシュボード。時計＋天気＋関東広域電車運行情報＋カウントダウン機能を統合。

---

## 1. 画面構成（縦並び）

```
┌──────────────────────────────────┐
│ [TrainWatcher] [日付] [🔋XX%]  ↺ │  ← ナビバー
├──────────────────────────────────┤
│   09:43:21                       │  ← 時計（中央大）
│ ─────────────                    │  ← 紫ライン
│ ※ ALLモード時のみ:               │
│   [カウントダウン・タイマー大]    │
│   [カウントダウン・サブ]          │
│                                  │
│ [NTP] [天気行：地点A/地点B/明日]  │
│  A / A / A / A- / C / 3          │  ← ランクサマリー
│ 地点A天気 / 関東広域電車情報 /   │  ← リンク行
│   モードセレクト(現モード) /      │
│   任意時刻 (CD/ALLモード時)       │
├──────────────────────────────────┤
│ 広域カード (件数・路線リスト)     │
│ 監視路線カード × N                │
└──────────────────────────────────┘
```

## 2. 時計

- 端末時計を 1秒ごと描画
- フォント: Nunito 900・letter-spacing 0.04em
- 色: `#D8DCE0`（クールシルバー）
- size: `clamp(2.8rem, 16vw, 4.5rem)`

## 3. 天気（Open-Meteo API）

### 表示2地点
- LOCATIONS 配列に `{name, label, lat, lon, color, elemId}` 形式で記述
- 各地点ごとに固有色を割り当て、天気行の駅名表示に反映

### 上の傘マーク（今日 / 明日）
| 確率 or コード | レベル | 表示 |
|---|---|---|
| heavy code (61-65,80-82,95-99) or P≥70 | 4 | 「☂️☂️ 傘必須！！」 赤 |
| P≥50 | 3 | 「☂️ 傘推奨！」 橙 |
| P≥30 | 2 | 「🌂 傘かも」 黄 |
| P≥20 | 1 | 「☁️ 予備」 水色 |
| P<20 | 0 | 「ー」 灰 |

- **今日傘マーク**: `daily.precipitation_probability_max[0]` ＋ `daily.weather_code[0]`（24h最大）
- **明日傘マーク**: `daily.precipitation_probability_max[1]` ＋ `daily.weather_code[1]`
- 各地点の高い方を採用、`/` で今日と明日を並列表示

### 各地点の天気行
- 「現在時刻」基準: アイコン・天気テキスト・気温・現在%（傘マーク判定とは別ソース）
- 明日行: 24h max/min気温・最大%

### 5分ごとに自動更新（`setInterval(fetchWeather, 5*60*1000)`）

## 4. 電車運行情報（Yahoo経由 Cloudflare Worker）

### プロキシ
- `https://trainwatch-proxy.s-tanaka-vega.workers.dev/?url=` 経由でYahoo CORSバイパス

### 対象路線（TARGET_LINES）
- HTML 内 `TARGET_LINES` 配列に固定。先頭は `'広域'` 固定、その後ろに監視したい路線名を任意で並べる
- 各路線の Yahoo 詳細ページ ID は `LINE_DETAIL_ID` で対応付け
- Yahoo 表記揺れの吸収は `TARGET_LINE_KEYMAP` で別名を登録

### 広域カード
- 関東他社全体（`skipKws` で重複表示する専用カード対象路線を除外）
- ヘッダ: `広域  遅延 N路線  !!`（件数色付き、!!バッジ左に件数）
- ヘッドラインのソート3層:
  1. 優先層: `priorityKws`（個人優先したい路線キーワード）を含む（五十音順）
  2. 中段: それ以外で status≠'その他'（五十音順）
  3. 最下段: status='その他'（五十音順）
- 件数バッジ色:
  | count | 色 | rank |
  |---|---|---|
  | 0 | 緑 `#34A853` | `-` |
  | 1〜3 | 黄 `#FBBC04` | `!` |
  | 4〜8 | 橙 `#FF6D01` | `!` |
  | 9〜15 | 赤 `#EA4335` | `!!` |
  | 16+ | 紫 `#9C27B0` | `!!` |

### ランクサマリー（広域カードの上、リンク行の上）
- 順序: TARGET_LINES の固定順（広域以外）
- 表示例:
  - 全平常 (count=0): `A / A / A / A- / C`
  - 関東他社のみ遅延（自分の優先路線は平常）: `A / A / A / A- / C / 9` （件数のみ）
  - 自分の優先路線が遅延中: `A / A / A / A- / C / 9 !!` （件数 + 重大度マーク）
- 末尾ルール:
  - count=0 → 末尾なし
  - count>0 かつ 自分の優先路線が平常 → 件数だけ
  - count>0 かつ 自分の優先路線が `A` 以外 → 件数 + `!`/`!!`
- 「自分の優先路線」は `userPriorityKws` で設定
- ランクと件数で同色付け、`/` グレー区切り

### 個別路線カード
- 詳細ページ（埋め込みツイート含む）から **遅延スコア・混雑スコア・遅延分数** 抽出
- ランク算出（calculateRank）:
  - 不通系（見合わせ・運休・脱線・全線ストップ等） → **E** 赤
  - 大幅・大規模・ダイヤ乱れ・遅延45分以上 → **D** 橙
  - その他の異常状態 → **C** 赤
  - 平常時のツイート由来:
    - tweetDelayMin≥45 → **D**
    - tweetHasStrong & ≥10分 → **C**
    - tweetDelayMin≥20 → **C**
    - delayScore≥8+offset or tweetDelayMin≥10 → **B**
    - delayScore≥3+offset or crowdScore≥3+offset → **A-**
    - それ以外 → **A** 緑
  - ラッシュ時 (7-10/17-20時) は閾値+3、深夜 (23-6時) は -2

### 5分ごとに自動更新（`setInterval(refresh, 5*60*1000)`）

## 5. カウントダウン

### 3つの表示モード（サイクル切替）
| モード | ボタン表示 | 表示内容 |
|---|---|---|
| weather | `モードセレクト(天気)` | NTP + 天気のみ |
| countdown | `モードセレクト(CD)` | カウントダウンのみ（天気・NTP非表示） |
| both | `モードセレクト(ALL)` | カウントダウン上 → NTP + 天気 下 |

- 切替: ボタンクリックで `天気 → CD → ALL → 天気` 循環
- `localStorage` (`trainwatch_display_mode`) に保存、次回開いた時に復元

### 自動スケジュール
- HTML 内 `COUNTDOWN_SCHEDULE` 配列に `{time: 'HH:MM', desc: '...'}` 形式で複数件埋め込み
- 1日の通勤・勤務イベントを並べておけばカウントダウンが自動的に進行
- 略記ルール例: `[路線略]`（路線名の頭文字）/ `[着]`（到着）など、好きな短縮表記を desc に書く

### 自動補充ロジック
- 現在時刻から最も近い未来の **2件**を `countdownTargets[]` に投入
- 期限超過から **60秒猶予期間** は表示維持（`0:00` 表示）
- 60秒経過したら自動削除 → 次の予定で補充

### 任意時刻ボタン（CD/ALLモード時のみ表示）
- 押下で `prompt` 起動（例: `1024 打合せ` / `14:30 出発`）
- 受付フォーマット: `HH:MM[:SS] [説明]` または `HHMM [説明]` / `HMM [説明]`
- 過去時刻入力時は翌日扱いに自動ロールオーバー
- 1件のみ保持（**セッション中限り**、リロードで消える）
- 入力時に天気モードなら自動的に ALL モードへ移行
- 上限超過時は最遠の自動枠を退避

### 表示
- メイン（最も近い1件）: ラベル `0.8em` + タイマー `1.65em` `font-weight:900`
- サブ（2件目）: ラベル `0.8em` + タイマー `1.25em` `font-weight:800`
- letter-spacing `0.04em`（時計と同じ「のびのび」感）
- フォント: 時計と同じ Nunito 系
- size: メイン `clamp(1.55rem, 5.6vw, 2.15rem)` / サブ `clamp(1.2rem, 4.4vw, 1.7rem)`

### 残り時間色（タイマー部のみ色変化、ラベルは中立）
| 残り | 色 |
|---|---|
| ≤1分 | 赤 `#EA4335` |
| ≤2分 | 橙 `#FF6D01` |
| ≤5分 | 濃い黄 `#F9A825` |
| ≤10分 | 水色 `#4FC3F7` |
| ≤30分 | 濃い青 `#1E88E5` |
| ≤60分 | 紫 `#AB47BC` |
| 60分超 | 濃い緑 `#388E3C` |

### 表示形式
- 予定（絶対時刻）: ゼロ詰め `09:43`
- 残り時間: ゼロ詰めなし `5:13` / `1:23` / `0:09`
- 1時間以上: `H:MM:SS`（例 `1:23:45`）

## 6. その他の機能

- 🔋 バッテリー: `navigator.getBattery()` で表示。≤20% 赤 / ≤50% 黄 / >50% シアン
- ↺ 手動更新ボタン: 天気 + 電車情報を即時再取得（読込中はスピナー）
- NTP/`未取得`/`OK`/`NG` インジケータ: 電車情報取得状況

## 7. オフライン挙動

| 機能 | オフライン |
|---|---|
| 時計 | ◯ |
| カウントダウン（自動・任意） | ◯ HTML内データのみで完結 |
| バッテリー | ◯ |
| モード記憶 (localStorage) | ◯ |
| 天気・電車情報 | × ネット必須 |
| Google Fonts | △ 初回キャッシュ後はOK |

GitHub Pages URL でも初回はネット必須。完全オフライン化したい場合は Service Worker (PWA化) が必要。

## 8. ファイル構成

- `index.html` 単体ファイル（CSS・JS全部内蔵）
- 外部依存:
  - 自前の Cloudflare Worker （Yahoo 運行情報用 CORS バイパスプロキシ）
  - Open-Meteo API（無料・無認証）
  - Yahoo 運行情報（プロキシ経由）
  - Google Fonts (Nunito)

---

## 9. 公開方法

- **個人情報内蔵のため、public 公開時は注意**:
  - スケジュール (時刻・場所略記)、地点座標、個人プロキシURL がベタ書き
  - 公開する場合は private repo or Cloudflare Pages + auth 推奨
- **ローカル運用** (iCloud Drive で `file://`) なら漏洩リスクなし
- localStorage に保存されるのは `trainwatch_display_mode` のみ。漏れても問題なし

## 10. 改造の出発点

### よく触りそうな箇所
- スケジュール変更 → `COUNTDOWN_SCHEDULE` 配列
- 監視路線追加 → `TARGET_LINES` + `LINE_DETAIL_ID` + `TARGET_LINE_KEYMAP`
- 天気地点変更 → `LOCATIONS`
- 色しきい値 → `colorOf()` (カウントダウン) / `umbrellaLevel()` `levelToAlert()` (傘) / `calculateRank()` (電車ランク)
- 表示モード追加 → `DISPLAY_MODES` + `MODE_LABEL_JP` + `applyDisplayMode()`
